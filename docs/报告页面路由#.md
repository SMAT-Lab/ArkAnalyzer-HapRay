## 单版本分析 - 步骤页面

### 格式：`#/step/{stepId}/{type}`

**注意**：`/step/{stepId}` 必须指定类型，不能单独使用。

| URL Hash | 页面 ID | 页面标题 | 说明 |
|----------|---------|----------|------|
| `#/step/1/perf` | `perf_step_1` | 步骤1负载分析 | 步骤1的负载分析 |
| `#/step/2/perf` | `perf_step_2` | 步骤2负载分析 | 步骤2的负载分析 |
| `#/step/{N}/perf` | `perf_step_{N}` | 步骤N负载分析 | 步骤N的负载分析 |
| `#/step/1/frame` | `frame_step_1` | 步骤1帧分析 | 步骤1的帧分析 |
| `#/step/{N}/frame` | `frame_step_{N}` | 步骤N帧分析 | 步骤N的帧分析 |
| `#/step/1/fault-tree` | `fault_tree_step_1` | 步骤1故障树分析 | 步骤1的故障树分析 |
| `#/step/{N}/fault-tree` | `fault_tree_step_{N}` | 步骤N故障树分析 | 步骤N的故障树分析 |
| `#/step/1/flame` | `flame_step_1` | 步骤1火焰图分析 | 步骤1的火焰图分析 |
| `#/step/{N}/flame` | `flame_step_{N}` | 步骤N火焰图分析 | 步骤N的火焰图分析 |
| `#/step/1/memory` | `memory_step_1` | 步骤1 Memory分析 | 步骤1的Memory分析 |
| `#/step/{N}/memory` | `memory_step_{N}` | 步骤N Memory分析 | 步骤N的Memory分析 |
| `#/step/1/ui` | `ui_animate_step_1` | 步骤1 UI 分析 | 步骤1的UI分析 |
| `#/step/{N}/ui` | `ui_animate_step_{N}` | 步骤N UI 分析 | 步骤N的UI分析 |
| `#/step/1/hilog` | `hilog_step_1` | 步骤1 日志分析 | 步骤1的日志分析 |

**无效路径**：
- ❌ `#/step/1` - 必须指定类型
- ❌ `#/step/2` - 必须指定类型

## 路径格式说明

### 路径结构

1. **基础页面**：`#/{pageName}`
   - 例如：`#/welcome`、`#/overview`、`#/summary`

2. **单版本步骤页面**：`#/step/{stepId}/{type}`
   - `{stepId}`：步骤编号（数字）
   - `{type}`：分析类型（`perf`、`frame`、`fault-tree`、`flame`、`memory`、`ui`、`hilog`）
   - **注意**：`/step/{stepId}` 必须指定类型，不能单独使用

3. **版本对比步骤页面**：`#/compare/step/{stepId}/{type}`
   - `{stepId}`：步骤编号（数字）
   - `{type}`：对比类型（`load`、`detail`、`new`、`top10`、`fault-tree`、`ui`）

4. **兼容格式**：`#/{pageId}`
   - 直接使用页面 ID 作为路径（用于旧格式兼容）

### 路径解析规则

- 空 hash 或 `#/` → 默认页面（`perf_load_overview`）
- 路径不区分大小写（但建议使用小写）
- 路径末尾的 `/` 会被自动去除
- 无效路径会返回 `null`，使用默认页面

### 示例

```javascript
// 基础页面
location.hash = '#/overview'        // → perf_load_overview
location.hash = '#/summary'         // → summary_overview

// 单版本步骤页面
location.hash = '#/step/1/perf'     // → perf_step_1
location.hash = '#/step/2/frame'    // → frame_step_2
location.hash = '#/step/3/fault-tree' // → fault_tree_step_3

// 版本对比步骤页面
location.hash = '#/compare/step/1/load'  // → compare_step_load_1
location.hash = '#/compare/step/2/detail' // → compare_step_detail_2

// 无效路径
location.hash = '#/step/1'          // → null (使用默认页面)
location.hash = '#/unknown'         // → null (使用默认页面)
```
